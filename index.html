Evidence-safe autonomous DFIR for Protocol SIFT.

SIFT Sentinel is a custom MCP server and self-correcting triage agent that lets an AI responder move quickly while preserving evidence, refusing unsupported claims, and producing traceable findings.

Typed MCP tools

The model does not receive arbitrary shell access. It calls structured forensic tools with validated inputs and predictable outputs.

  • Memory process and network review
  • Prefetch, Amcache, timeline, and events
  • Registry Run key analysis

Evidence policy

Evidence paths are read-only by code. Generated reports, logs, and manifests are written only under case outputs.

  • Pre-run manifest
  • Post-run manifest
  • Spoliation test command

Self-correction

The agent starts with inferred leads, requests missing checks, and updates findings to confirmed or refuted.

  • Validation gaps logged
  • Tool call IDs preserved
  • Weak Prefetch-only lead refuted

Autonomous triage workflow

Every finding remains tied to evidence. Every run remains auditable.

1. InventoryHash configured case evidence and record artifact metadata.
2. TriageRun typed memory, disk, event, and registry tools.
3. ValidateIdentify missing corroboration before confirming claims.
4. CorrectConfirm supported findings and refute unsupported leads.
5. ReportWrite JSONL logs, findings, accuracy score, and integrity proof.

SIFT Workstation integration

The static demo runs anywhere with Python. The same policy layer supports SIFT wrapper contracts for Volatility 3, EvtxECmd, MFTECmd, PECmd, AmcacheParser, RECmd, YARA, and Sleuth Kit.
